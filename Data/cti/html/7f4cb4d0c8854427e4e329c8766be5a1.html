CB TAU Threat Intelligence Notification: Emotet Utilizing WMI to Launch PowerShell Encoded Code

Swee Lai Lee
April 24, 2019

Carbon Black recently learned that a customer had received a malicious email, which was written in German and was attached with a password-protected zip file that contained a malicious document file. This phishing email belongs to the recent Emotet campaign. However, what makes this malware unique is that the document contains malicious VBA code that uses WMI (winmgmts:win32_processStartup) to launch PowerShell with encoded code. This, in turn, downloads a second payload which is the Emotet banking trojan to compromise the victim’s machine. Detailed analysis on the Carbon Black User Exchange can be foundhere.

(You may also find another recent Emotet campaign threat notificationhere.)

Behavior Summary

Upon execution, the malicious VBA code in the document will utilize WMI to launch powershell encoded code and then download the second stage payload, named as 615.exe, which is the Emotet banking trojan. Thus, the parent process of Wmiprvse.exe will be svchost.exe but not winword.exe.

The downloaded malicious payload 615.exe will then further compromise in the victims computer. It will copy and renamed itself to the path c:\windows\syswow64\repstarta.exe to continue its persistence in the computer.
